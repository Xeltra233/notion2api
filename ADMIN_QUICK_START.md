# Admin Quick Start

<a id="admin-quick-start"></a>

这是一份从部署到进入后台、再到验证 Runtime 保存的最短路径文档。

## 1. 准备最小配置

先复制环境变量模板：

```bash
cp .env.example .env
```

最小可用配置：

```env
ADMIN_PASSWORD=change-me-now
HOST=0.0.0.0
PORT=8000
```

说明：

- `ADMIN_PASSWORD`：后台登录密码
- `HOST` / `PORT`：服务监听地址
- 如果你暂时只是先把后台跑起来，不需要一开始就准备账号池

默认后台账号：

- 用户名：`admin`
- 密码：你设置的 `ADMIN_PASSWORD`

## 2. 启动服务

### 本地运行

```bash
pip install -r requirements.txt
uvicorn app.server:app --host 0.0.0.0 --port 8000
```

### Docker

```bash
docker-compose up -d
```

## 3. 验证服务已经起来

检查健康接口：

```bash
curl http://127.0.0.1:8000/health
```

预期至少应该返回：

```json
{"status":"ok"}
```

## 4. 打开后台入口

直接访问根路径：

- 本地：`http://127.0.0.1:8000/`
- 线上：你的部署域名根路径

页面默认先显示后台登录页。

## 5. 登录后台

默认只需要填：

- `后台用户名`
- `后台密码`

然后点击：`登录后台`

### 什么时候要展开“连接设置”

只有这两种情况才需要：

- 你要连另一个实例
- 当前实例要求 API Key，后台登录被拦住

连接设置里有：

- `Base URL`
- `访问密钥`

如果后台登录因为 API Key 失败，前端会自动展开“连接设置”。

## 6. 判断是否真的进入后台

登录成功后，应该看到这些模块已解锁：

- `总览`
- `账号`
- `运行时`
- `诊断`
- `用量`

如果你仍然只停在 `登录` 模块，或者其它模块点不开，就说明后台并没有真正进入成功。

## 7. 第一次进入后台后先做什么

推荐顺序：

1. 先看 `总览`
2. 再进 `运行时`
3. 确认基础配置是否符合当前部署环境
4. 如需导入账号，再进 `账号` / `诊断`

## 8. 修改并保存 Runtime 配置

进入：`运行时`

建议先改一个容易识别的字段，例如：

- 探测间隔
- Chat 开关
- 代理模式
- 自动注册相关配置

改完后点击：`保存配置`

保存后前端会做两件事：

- 在 Runtime 模块内部显示保存结果提示
- 立即重新从后端加载一次运行时配置

这一步就是为了避免“看起来像保存了，但其实只是前端表单变了”的假象。

## 9. 验证 Runtime 保存是否真的生效

按这个顺序检查：

1. 修改 Runtime 配置
2. 点击 `保存配置`
3. 观察 Runtime 模块里的保存提示
4. 再点一次 `加载配置`
5. 刷新页面
6. 重新进入 `运行时`
7. 确认刚才改的值仍然存在

只有刷新后值还在，才算真正保存成功。

## 10. 常见问题

### 修改 Chat password 后旧聊天会话为什么失效了

这是当前预期行为。

为了避免后台修改聊天密码或关闭聊天密码后，旧的聊天访问 session 继续可用，服务现在会在以下场景主动清空现有 chat session：

- 修改 Chat password
- 清空 Chat password
- 仅切换 Chat password enabled 开关

如果你刚改完配置，原先已经打开的聊天页需要重新登录一次聊天访问。

### 为什么邮箱登录 / 自动注册的代理行为要看代理模式

当前后台里的邮箱验证码登录、以及自动注册流程，都会跟随 Runtime 中的当前代理模式，而不只是读取单独的 `upstream_proxy` 文本框。

如果你使用的是这些模式之一：

- `warp`
- `socks5`
- `http`
- `https`

请优先确认 Runtime 里的代理模式和对应代理地址都已经保存成功，再去测试邮箱登录或自动注册。

### 后台看起来像登录了，但什么都做不了

这通常是浏览器里残留了失效的 `admin session`。

当前前端会在后台校验失败时：

- 清掉失效 session
- 回退到登录态
- 不再保留假的“已登录后台”界面

### 点了登录后 Runtime 还是打不开

优先排查：

- 当前实例是否要求 API Key
- `Base URL` 是否填对
- `访问密钥` 是否有效

### 点了保存没提示

现在保存结果会直接显示在 Runtime 模块内部。

如果仍然没有变化，先确认：

- 当前确实已经进入后台
- 保存请求没有被 API Key 或 session 拦住

### 刷新后配置又回去了

优先检查：

- 保存请求是否真的成功
- 当前连接的是不是你以为的那个实例
- 当前实例是否有权限把配置写回运行时文件

## 11. 推荐的最短操作路径

如果你只想最快从部署走到可用，按这条线：

1. 配 `.env`
2. 启动服务
3. 打开 `/`
4. 登录后台
5. 进 `运行时`
6. 保存一项配置并刷新确认
7. 再去 `账号` / `诊断` 做后续导入和排查
