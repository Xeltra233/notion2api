# Admin Runtime Notes

<a id="admin-access-runtime-notes"></a>

## 后台访问与 Runtime 保存

- 登录页默认只正面展示后台用户名和密码
- `Base URL` 与访问密钥被收进“连接设置”，只有连接别的实例，或者当前实例要求 API Key 时才需要展开
- 如果后台登录因为缺少或错误的 API Key 被拦住，前端会自动展开“连接设置”并提示补录访问密钥
- Runtime 页点击“保存配置”后，结果会直接显示在 Runtime 模块内部，而不只依赖顶部全局提示
- Runtime 保存成功后，前端会立即重新从后端加载一次运行时配置，避免出现“前端看起来像已保存，但刷新后又回去”的假象

## 建议测试顺序

1. 先确认后台已经成功登录，`总览`、`账号`、`运行时`、`诊断`、`用量` 模块都已解锁
2. 如果后台登录失败且提示与 API Key 相关，先展开“连接设置”补录访问密钥
3. 进入 `运行时` 修改配置后点击“保存配置”
4. 观察 Runtime 摘要卡内的保存结果提示
5. 如需确认持久化结果，再点击“加载配置”或直接刷新页面复查
