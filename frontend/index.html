运行时

把服务调到你想要的状态

这里主要处理服务怎么跑、Chat 要不要开放、代理怎么接、自动注册是否启用。第一次使用时，不需要一次填完全部内容。

你通常会先在这里做什么
先决定是否启用 Chat、是否需要单独聊天密码，再补 API Key、代理和自动化设置。复杂项可以晚一点再配。
更轻的起步方式
通常只要先准备 `ADMIN_PASSWORD` 和端口就够了。账号、`API_KEY`、代理、自动注册、工作区和 refresh 这些更细的内容，后续再在后台里慢慢补齐。
基础设置
首次使用时优先关注这些设置。`ALLOWED_ORIGINS` 属于启动期配置，修改后通常仍需重启服务。
聊天入口
这里决定聊天模块要不要在左侧出现，以及进入聊天前是否要单独再过一道密码。
更多运行时能力
下面这些能力按场景拆开了。只展开你现在真正要改的那一块，不需要一次把所有字段都看完。
代理与上游网络
当服务需要走 HTTP、HTTPS、SOCKS5 或 Warp 时，在这里补齐完整网络出口。
工作区与 refresh
只有在要联动模板、外部 refresh 接口或真实工作区创建流程时，才需要改这里。
自动注册
只有在你希望系统自己补新号、补邮箱或定时拉起注册动作时，才需要打开这块。
媒体与公开链接
这块是给图片缓存和对外可访问链接预留的位置。即使现在还没完全接入媒体上传流程，也先把入口放正，避免以后继续塞进杂项里。
下一批重点
配置 public base URL 后，新上传图片会直接生成这个公开地址下的链接；不填时，默认使用当前服务自己的 `/v1/media/...` 地址。媒体缓存目录建议指向持久化挂载目录，避免服务重启后图片失效。
保存运行时配置后，会立即更新当前服务设置。多数运行时参数会直接生效；`ALLOWED_ORIGINS` 这类启动期配置通常仍需重启。Warp 本地代理可填写 warp-yg 等工具暴露的端口。媒体 public base URL 不填时，聊天图片会自动跟随当前服务地址生成 `/v1/media/...` 链接。
