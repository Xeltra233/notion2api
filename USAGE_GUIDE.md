# Notion AI 使用场景指��

## 🎯 推荐使用场景

### ✅ 强烈推荐

#### 1. **对话式聊天（Cherry Studio, ChatBox 等）**
- **场景**: 日常对话、代码助手、写作辅助
- **推荐模型**: `claude-sonnet4.6`（智能+速度平衡最佳）
- **预期响应**: 3-5秒/次
- **体验**: ⭐⭐⭐⭐⭐

#### 2. **短文本翻译（Zotero 插件等）**
- **场景**: 论文摘要翻译、段落翻译
- **推荐模型**: `claude-sonnet4.6` 或 `claude-opus4.6`
- **预期响应**: 2-4秒/次
- **体验**: ⭐⭐⭐⭐⭐

#### 3. **单轮问答 API 调用**
- **场景**: 翻译 API、摘要生成、文本处理
- **推荐模型**: `claude-sonnet4.6`
- **预期响应**: 3-5秒/次
- **体验**: ⭐⭐⭐⭐

---

## ⚠️ 不推荐使用

### ❌ **沉浸式翻译（全页面翻译）**

#### 为什么不推荐？
1. **速度问题**
   - Notion AI 响应时间：3-5秒/次
   - 沉浸式翻译需要处理整个页面，可能需要数十次调用
   - 总耗时：3-5秒 × 20-50次 = 1-4分钟

2. **限流问题**
   - Lite 模式：30次/分钟
   - Heavy 模式：20次/分钟
   - 大页面翻译容易触发限流

3. **体验问题**
   - 用户需要等待过长时间
   - 容易超时或失败
   - 成本高（每次调用都消耗 Notion 配额）

#### 💡 推荐替代方案
| 需求 | 推荐方案 | 原因 |
|------|---------|------|
| 全页面翻译 | DeepL、Google 翻译 | 速度快，专门优化 |
| 技术文档翻译 | 沉浸式翻译 + DeepL API | 速度快，质量好 |
| 论文翻译 | Zotero + Notion AI | 段落级翻译，体验好 |

## 🚀 性能优化建议

### 1. **选择合适的模型**

| 模型 | 速度 | 质量 | 适用场景 |
|------|------|------|---------|
| `claude-sonnet4.6` | ⭐⭐⭐⭐ | ⭐⭐⭐⭐ | **默认推荐**，平衡速度和质量 |
| `claude-opus4.6` | ⭐⭐⭐ | ⭐⭐⭐⭐⭐ | 复杂推理、长文本、高质量翻译 |
| `gemini-3.1pro` | ⭐⭐⭐ | ⭐⭐⭐ | 备用方案 |
| `gpt-5.2` / `gpt-5.4` | ⭐⭐ | ⭐⭐⭐ | 不推荐（速度慢） |

### 2. **Lite vs Heavy 模式选择**

```
┌─────────────────────────────────────────────────────────────┐
│ 选择 Lite 模式：                                             │
│ ✅ 翻译服务（段落级）                                         │
│ ✅ 单轮问答 API                                              │
│ ✅ 水平扩展部署                                              │
│ ✅ 无需会话记忆                                              │
├─────────────────────────────────────────────────────────────┤
│ 选择 Heavy 模式：                                            │
│ ✅ 长对话聊天                                                │
│ ✅ 需要上下文记忆                                            │
│ ✅ 多轮对话任务                                              │
│ ✅ 本地部署                                                  │
└─────────────────────────────────────────────────────────────┘
```

### 3. **速率限制优化**

#### 当前设置：
- **Lite 模式**: 30次/分钟
- **Heavy 模式**: 20次/分钟

#### 自定义调整：
编辑 `app/limiter.py`:
```python
# 更激进的限制（适合小规模使用）
if is_lite_mode():
    default_limit = "60/minute"  # 60次/分钟

# 更保守的限制（避免过度消耗配额）
if is_lite_mode():
    default_limit = "15/minute"  # 15次/分钟
```

### 4. **缓存建议**（未来功能）

如果需要加速重复请求，建议添加：
- Redis 缓存层
- 本地文件缓存
- CDN 缓存（翻译场景）

## 🔧 故障排查

### 问题：响应太慢
**解决方案：**
1. 切换到 `claude-sonnet4.6`
2. 检查网络连接
3. 减少 prompt 长度

### 问题：频繁限流
**解决方案：**
1. 编辑 `app/limiter.py` 提高限制
2. 使用 Lite 模式（30次/分钟）
3. 添加客户端缓存

### 问题：沉浸式翻译超时
**解决方案：**
1. **改用 DeepL 或 Google 翻译**（强烈推荐）
2. 如果坚持用 Notion AI：
   - 降低并发度
   - 分段翻译
   - 增加超时时间
